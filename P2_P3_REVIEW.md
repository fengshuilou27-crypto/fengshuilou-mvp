## 1. 修復詳情

### 1.1 P0: 雙人模式 500 錯誤 ✅

**根因：** `app.py` 調用 `analyze_bagua_dual` 時使用位置參數：`weight_a` 被傳到 `room_positions` 位置，`weight_b` 被傳到 `goal` 位置。

```python
# 修復前（錯誤）
bagua_dual = analyze_bagua_dual(
    meta.birth_date, meta.user_gender,
    meta.cohabitant_birth_date, meta.cohabitant_gender,
    meta.building_facing, weight_a, weight_b  # ❌ weight_a 被當作 room_positions
)

# 修復後（正確）
bagua_dual = analyze_bagua_dual(
    meta.birth_date, meta.user_gender,
    meta.cohabitant_birth_date, meta.cohabitant_gender,
    meta.building_facing,
    weight_a=weight_a, weight_b=weight_b  # ✅ 關鍵字參數
)
```

**驗證：** 雙人模式測試通過，得分 72.1，八宅對照表正常。

### 2.2 關鍵改進分析

#### 改進 A：雙人模式 500（決定性提升）

**修復前：** 用戶未選目標 → 目標得分 0，顯示「未選擇任何目標」，總分被嚴重低估。

**修復後：** 無目標時自動使用「財富」並提示用戶，總分正常計算。

**這是算法容錯性的重要提升。**

#### 改進 C：化解建議去重（專業可信度）

**修復後：** 建議去重，最多每條建議出現一次。
